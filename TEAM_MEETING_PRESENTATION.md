## SLIDE 5: Your Tools - GitHub

**Title:** GitHub - Version Control

**What It Does:**
- Stores all game files in the cloud
- Tracks every change with full history
- Lets us work on different parts simultaneously
- Automatic backup of everything

**How You'll Use It:**
1. Clone the repository (one-time setup)
2. Before working: Run `pull-updates.bat`

**Suggested Visual:**
Git branching diagram or GitHub logo

**Speaker Notes:**
GitHub is our central hub. It's like Google Docs but for game development - it tracks every change, who made it, and when. We've created simple scripts so you never need to touch the command line. Just double-click pull-updates before you start work, and quick-commit when you're done.

## SLIDE 12: Coordination Rules ‚ö†Ô∏è

**Title:** Avoiding Conflicts

**The Golden Rules:**

üö® **ONE person in RPG Maker at a time**

‚úÖ **What's Safe to Do Simultaneously:**
- Family 1: Edit maps/events
- Family 2: Create NEW art assets
- Anyone: Write docs/story

‚ùå **What Causes Conflicts:**
- Two people editing same map
- Two people editing database
- Two people editing same art file

**Always communicate in Discord first!**

**Suggested Visual:**
Warning triangle and traffic light (red/green)

**Speaker Notes:**
This is critical. RPG Maker MZ doesn't support multiple people editing at once. If two people change the same map, one person's work gets lost. Always coordinate in Discord. Family 1 can work on game logic while Family 2 creates new sprites - that's fine. But if you want to edit an existing map or character, make sure nobody else is working on it.
